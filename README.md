# Churn-risk-segmentation-dashboard
End-to-end churn analysis combining SQL, Random Forest, and Power BI to identify and monitor high-risk customer segments.

This project started as an attempt to understand why customers leave telecom services and how we can predict it early. The focus was not just building a model, but connecting it with a dashboard that actually explains the “why” behind churn.

## What I worked on
- Cleaned and transformed raw customer data using SQL
- Explored patterns like tenure, contract type, and service usage
- Built a Random Forest model to classify churn vs non-churn customers
- Exported predictions and connected them to Power BI
- Designed a dashboard to make insights easy to interpret

## Some observations
- Customers with month-to-month contracts showed noticeably higher churn
- Short tenure users (under 1 year) were more likely to leave
- Certain services like fiber internet had higher churn compared to others
- Payment methods also showed small but consistent patterns

## Dashboard highlights
- Churn rate overview and customer distribution
- Breakdown by contract type, internet service, and tenure
- Prediction view showing high-risk customers
- Simple layout focused on readability over heavy design

 ## Final note
* This project helped me understand how to go beyond just building a model and actually connect it to business insights, which is something I’m continuing to improve.
